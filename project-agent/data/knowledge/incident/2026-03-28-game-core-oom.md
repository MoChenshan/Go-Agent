
# 故障复盘：game-core 2026-03-28 全服重启事故

## 一、事件概要

| 项目 | 内容 |
|------|------|
| 发生时间 | 2026-03-28 19:12 ~ 19:47 (CST) |
| 影响范围 | 全国区玩家无法登录，持续 35 分钟 |
| 严重等级 | P1 |
| 根因 | game-core v1.4.2 发版后 OOM，全集群 CrashLoopBackOff |

## 二、时间线

- **19:08** 运维同学通过蓝盾触发 game-core v1.4.2 全量发布
- **19:12** 蓝鲸监控告警：game-core Pod 异常重启率 > 80%
- **19:15** 值班同学介入，kubectl 查看发现 OOMKilled
- **19:22** 定位到 v1.4.2 新增的离线任务初始化内存峰值 4.2GB，limit 仍为 2GB
- **19:40** Helm rollback 到 v1.4.1
- **19:47** 所有 Pod Ready，服务恢复

## 三、根因分析

新版本在 `init()` 中加载了全量玩家库存缓存（为了后续查询优化），但：
1. 内存估算时漏算了装备多维度索引
2. 压测环境是小样本，未暴露问题
3. 发版前 canary 只观察了 5 分钟，init 还未完全触发

## 四、改进项

- [x] game-core limit 上调至 6GB
- [x] 初始化缓存改为懒加载
- [x] canary 观察时间延长至 15 分钟
- [ ] 引入 pre-release 内存画像工具
- [ ] 建立发布前 `kubectl top` 对比机制

## 五、GameOps Agent 可提供的帮助

- 告警首响：秒级定位到 game-core 服务 + Pod 异常重启
- 历史案例检索：关键词 "OOM" / "init 内存" 能检索到本复盘
- 一键回滚：输出 Helm history 并请求确认后回滚到 v1.4.1
