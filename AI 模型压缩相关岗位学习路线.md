# AI 模型压缩与推理部署——综合学习路线

> **定位**：以「大模型推理部署工程师」为主攻方向（当前需求量最大、薪资天花板最高），以「模型压缩算法」为核心差异化竞争力，辅以必要的工具链知识。一条路线打通三个岗位的核心能力。

> **为什么选推理部署为主线？**
> - 🔥 大模型落地是 2024–2026 行业最大痛点，几乎所有大厂/AI 公司都在高薪抢人
> - 💰 推理部署直接关联降本增效（推理成本占大模型 TCO 的 80%+），业务价值最直接
> - 🧩 推理部署天然需要压缩技术（量化是标配），学一个方向等于覆盖两个岗位
> - 🛠️ 工具链知识（ONNX/计算图）是推理部署的日常基本功，自然融入

## 阶段四：实战项目 + 面试冲刺（6~8 周）

### 4.1 必做项目（4 周，选 2 个即可覆盖面试）

#### 项目 A：LLM 推理服务搭建（主线项目，必做）

```
LLaMA-7B/13B → GPTQ 4-bit 量化 → vLLM/TensorRT-LLM 部署
→ 支持 Continuous Batching + KV Cache + 多并发
→ 输出：首Token延迟、生成吞吐(tokens/s)、显存占用、QPS
```

#### 项目 B：多卡分布式部署（加分项）

```
70B 模型 → 张量并行(TP=4) + 量化(INT4/FP8) → 多卡推理
→ 优化通信瓶颈，对比不同并行策略
→ 输出：端到端延迟、通信开销占比、扩展效率
```

#### 项目 C：CV 模型压缩全流程（补充项，展示压缩全栈能力）

```
YOLOv8 → 结构化剪枝(50% FLOPs) → INT8 量化 → TensorRT 部署
→ 输出：速度提升X倍、mAP损失<1%、内存降低X%
```

> ⚠️ **关键提醒**：所有项目必须体现**量化成果**（速度提升X倍、内存降低X%、精度损失控制在X以内），不能只讲原理。

### 4.2 面试复习冲刺（2~4 周）

#### 第一轮：基础 + 原理（3~5 天）

- [ ] 手推核心公式：量化公式（scale/zero_point 推导）、Softmax 数值稳定性、交叉熵梯度
- [ ] 默写 Transformer 架构：MHA 计算流程、KV Cache 在推理时的工作方式、RoPE 公式
- [ ] 编程题：LeetCode 中等每日 2 题（数组/链表/树/DP），重点刷字节/腾讯高频题

#### 第二轮：专项深挖（3~5 天）

- [ ] 量化专项：PTQ vs QAT 全面对比、GPTQ 的 Hessian 近似推导、AWQ 激活感知原理、SmoothQuant 数学推导
- [ ] 推理优化：KV Cache 显存计算公式（`2 × n_layers × n_heads × head_dim × seq_len × 2bytes`）、FlashAttention IO 复杂度、TP/PP 通信量公式
- [ ] 系统设计：大模型推理服务全链路（负载均衡 → 请求调度 → 模型推理 → 流式输出）

#### 第三轮：项目复盘（3~5 天）

- [ ] 每个项目准备 STAR 描述（Situation → Task → Action → Result）
- [ ] 量化成果数据烂熟于心（速度提升X倍、内存降低X%、精度损失X%）
- [ ] 准备 2 个技术深坑故事（踩坑 → 排查过程 → 解决方案 → 经验沉淀）

#### 第四轮：系统设计 + 开放题（3~5 天）

- [ ] 设计题：设计一个支持百万并发的大模型推理系统
- [ ] 设计题：设计一个自动化模型压缩 pipeline（输入约束 → 自动搜索最优策略）
- [ ] 前沿话题：MoE 推理优化、FP8 生态、端侧大模型（手机/边缘设备）、多模态 VLM 推理
