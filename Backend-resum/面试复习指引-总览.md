# 面试复习指引 - 总览

> 本文档整理面试准备资料，将需要复习的内容与项目文档/代码关联起来，方便针对性复习。

## 复习策略

### 1. 优先级原则
- **🔴 最高优先级**：简历中标注的技术亮点，必须能讲清楚 why/what/how
- **🟡 高优先级**：面试官可能追问的延伸知识点
- **🟢 中优先级**：了解即可，能回答基本概念

### 2. 复习方法
- **看代码**：重读项目代码，能画出架构图/流程图
- **看文档**：重读项目文档，特别是 INTERVIEW.md 和架构设计文档
- **看报告**：重读压测报告、性能报告，能解释数据来源
- **补短板**：对不熟悉的知识点，查阅外部资料补充

## 项目一：游戏服务端（传统后端岗）

### 业务背景
- **游戏类型**：MMO（大型多人在线游戏）
- **玩家规模**：数十万在线，核心接口 QPS 十万级
- **核心玩法**：农场玩法（社交玩法，DAU 10万+），玩家种植/收获/餐厅经营/社交互动
- **痛点**：高并发场景下的数据一致性、缓存穿透、消息幂等性

### 技术亮点与复习资料

#### 🔴 协程模型（最高优先级）
- **简历描述**：基于协程框架实现数据库异步读写，将阻塞 I/O 转为协程挂起
- **面试问题**：协程 vs 线程？N:M模型？park/unpark原理？

  - [ ] 外部资料：[Java Project Loom 官方文档](https://openjdk.org/projects/loom/)（了解协程发展趋势）

#### 🔴 分布式锁（最高优先级）

## 通用面试准备

### 1. 自我介绍（1-2分钟）
- 背景：1-3年后端开发，腾讯游戏服务端 + AI Agent基础设施
- 技能：Go/Java双语栈，分布式业务后端 + AI Agent/LLM工程
- 项目：3个项目（游戏服务端、GameOps Agent、LLM微调推理）
- 优势：既能做传统业务后端，也能做AI应用工程

### 2. 项目介绍（每个项目3-5分钟）
- **项目一**：MMO游戏服务端，60+微服务，负责农场玩法，解决高并发数据一致性问题
- **项目二**：GameOps Agent，多Agent运维系统，解决告警处理效率低问题
- **项目三**：LLM微调推理，解决NPC对话角色一致性和运维知识库幻觉率问题

### 3. 技术亮点准备（每个亮点能讲5分钟）
- 协程模型与高并发处理
- 分布式锁与一致性设计
- Multi-Agent编排架构
- RAG知识库检索优化
- LLM三阶段训练流程
- vLLM推理优化实践

### 4. 踩坑故事准备（每个故事能讲3分钟）
- vLLM V1切换后prefix-cache命中率下降（项目二）
- 分布式锁续期失败导致死锁（项目一）
- DPO训练reward设计不合理导致模型退化（项目三）

## 注意事项

1. **诚实原则**：简历中标注"预期""方案设计"的内容，面试时主动说明，不要假装已完成
2. **深度优先**：选3-5个核心点深入准备，能讲清楚why/what/how，而不是面面俱到
3. **准备故事**：每个项目准备1-2个"踩坑/优化"故事（STAR模式）
4. **代码为王**：能画架构图/流程图，能解释代码实现细节，比背概念更重要
