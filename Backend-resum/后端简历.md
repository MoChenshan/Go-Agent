### 项目二：GameOps Agent — 多 Agent 智能运维系统（Go + LLM + RAG）

**技术栈**: Go · trpc-agent-go · MCP 协议 · Redis · Langfuse · OTel · Qwen3-8B

**项目概述**: **主导设计并开发**基于 Go 的多 Agent 协作运维系统，解决运维团队**告警处理效率低、故障诊断依赖人工经验**的痛点。系统实现从**告警接收（服务实例 CPU/内存异常）→ 故障诊断（日志分析+指标关联）→ 修复建议（重启/扩容/回滚）**的端到端智能运维流程。系统包含 Coordinator + 4 个子 Agent（Diagnosis/Repair/Knowledge/FileAnalyst），集成 **MCP 工具协议**接入蓝鲸监控/BCS 等运维平台；通过 **MCP 协议接入独立的 RAG 知识库服务**（含运维文档、故障处理 SOP、历史案例）为 Agent 提供运维知识支撑；支持 **HITL 人机协作审批**（高风险操作需人工确认）和**操作审计**，全链路接入 OTel GenAI v1.30 + Langfuse 可观测。

**主要工作**:

- **多 Agent 编排架构设计**: 设计 Coordinator + 子 Agent 的多级编排架构，Coordinator 负责意图路由（如"CPU 异常告警"路由至 Diagnosis Agent），子 Agent 各自维护独立 system prompt + 工具白名单。通过**工具可见性切片**实现权限隔离（Diagnosis Agent 只见只读工具，Repair Agent 只见写工具），**从物理层面杜绝工具越权**。子 Agent 间禁止互相调用，保证审计日志为线性树结构，便于故障复盘
- **RAG 知识库集成**: 通过 **MCP Streamable-HTTP 协议**接入独立 RAG 知识库服务（Agent 侧零代码变更）。RAG 服务采用 **BGE-M3 混合检索**（dense+sparse 一次前向输出，对运维场景的命令名/告警码等 lexical-heavy 查询提升显著）+ **RRF 融合**（k=60）+ **BGE-Reranker-v2-m3 精排** + MMR 多样化去重。citation 覆盖率从 60% 提升至 92%，幻觉率降低 10pp（微调模型 + 推理优化联合效果）
- **异步任务系统与 HITL**: 设计生产级异步任务系统，将 30 分钟级长任务（如日志分析、批量重启）从同步阻塞改为**Submit→Poll→Wait 异步语义**，实现 6 状态状态机（pending/running/succeeded/failed/cancelled/timed_out）+ 幂等去重。HITL 采用两段式确认框架——写操作（重启/扩容）通过 SSE interrupt 事件中断流式输出，等待用户 approve/reject 后继续。**避免 AI 误操作导致线上事故**。设计操作审计链保障可追溯
- **成本控制与可观测**: 实现 **Token 会计**（按字符类别分桶估算，纯 Go 实现不引入 cgo，误差 ±10%）+ **模型路由**（5 条短路规则：预算耗尽→cheap / 长 prompt→premium 等）+ **Prompt Cache**（LRU+TTL，SHA-256 指纹做 key，命中直接跳过 LLM 调用）。接入 **OTel GenAI Semantic Convention v1.30**，配合 Prometheus + Grafana 搭建可观测性体系，**降低 API 成本约 30%**
- **Prompt 工程与优化**: 设计三层 prompt 体系——System 层（含 Severity 自动升级规则）+ 工具描述层（每个工具写明"何时调用/何时不调用/失败 fallback"）+ Few-shot 层（含消极示例防 Coordinator 抢答）。踩坑：vLLM V1 切换后 prefix-cache 命中从 78% 暴跌至 30%，定位为 system prompt 嵌入动态时间戳，**将固定段前置 + 时间戳移至 user 段后命中恢复至 78%**

### 项目三：LLM 微调与推理部署工程

**技术栈**: Python · Qwen3-8B/4B · LLaMA-Factory · QLoRA · DPO · GRPO · vLLM · FP8 · EAGLE-3 · DeepSpeed ZeRO-3 · Liger Kernel

**项目概述**: 主导完成 LLM 微调（SFT/DPO/GRPO）与推理部署（vLLM + FP8 + EAGLE-3 投机解码）全链路工程。解决**AI 角色对话角色一致性差（原模型 88%→96%）、运维知识库问答幻觉率高（原模型 25%→15%）**两个问题。覆盖数据合成、三阶段训练、推理优化、评测体系搭建，全链路接入 Langfuse + Prometheus 观测。模型服务于**AI 角色对话**（用户与AI角色对话，需角色一致性）与**运维知识库问答**（运维人员查询故障处理方案，需准确性）两类场景。

**主要工作**:

- **三阶段训练全流程**: 完成 **SFT（QLoRA 4bit + Liger Kernel 加速 + NEFTune 噪声正则 + neat_packing 训练速度+35%）→ DPO（双温度采样 0.3/1.0 + 异源 Judge 构造偏好对，β=0.1）→ GRPO（5 种可验证规则 reward：format/scenario/action/length/role，组内归一化省掉 Critic）** 三阶段训练方案设计与配置，含完整 reward 函数实现。AI角色一致性提升 8pp（88%→**96%**），操作指令 JSON 可解析率达 99%（72%→**99%**）；知识库模型 RAGAS 评测 faithfulness 提升 17pp（0.72→**0.89**）
- **推理优化四档压测**: 实现 vLLM V0→V1→V1+FP8→**V1+FP8+EAGLE-3 四档实测**，**端到端吞吐 3.67× 加速**（45→165 tok/s），**TTFT(首token延迟) 从 280ms 降到 65ms**。配置 prefix-cache 后 **KV-cache 命中率 42%→78%**，FP8 量化模型单卡部署显存占用约 12GB。生产部署 FP8 为主、GPTQ-Marlin 为备（H100 上 FP8 精度更高，A10 上 Marlin kernel 速度反超 FP16）**，单卡支撑 96 并发**
- **数据工程与评测体系**: 实现 **DeepSeek-V3.2 + Magpie 双源合成数据管线**，BGE-M3 语义去重（cosine > 0.95 视为重复）+ RAGAS faithfulness 过滤**去掉部分低质量数据**。实现 PII 脱敏（正则 + NER 二次过滤）+ **对抗样本构造**（越权/虚假前提占 5%）。评测搭建三层：**领域金标**（100 条人工标注，每季度轮换防过拟合）+ **RAGAS**（context_precision/recall/faithfulness）+ **LLM-as-Judge 三维打分**（AnswerCorrectness/EvidenceSufficiency/ToolSelectionAccuracy，**异源 Judge + 双向打分校准 Position bias**），CI 门禁任一维度低于阈值阻断合并。**数据飞轮**：周级节奏自动采样坏case → 半自动标注 → 增量训练 → 离线评测 → 灰度上线，加速模型迭代周期
- **分布式训练实践**: 跑通 **DeepSpeed ZeRO-2/3 分布式训练**（8B QLoRA 显存 19.2GB→**9.5GB**，含 NVMe Offload），了解分布式训练基本概念，加速模型迭代周期
