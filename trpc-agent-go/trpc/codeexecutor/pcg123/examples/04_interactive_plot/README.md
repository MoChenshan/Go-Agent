# 🎨 交互式图形绘制示例

**演示** 如何在交互式模式下进行复杂的数据可视化，利用状态保持能力构建高级图表。

## 展示内容

- 🎨 **渐进式绘图**: 分步骤创建复杂的数据可视化
- 📊 **多阶段分析**: 数据加载 → 处理 → 分析 → 可视化
- 🖼️ **多图表生成**: 在同一会话中生成多个相关图表
- 💡 **智能复用**: 利用交互式状态避免重复计算

## 核心代码

```go
// 创建交互式执行器，优化图形生成参数
executor, cancel, err := pcg123.NewCodeExecutor(conf,
    pcg123.WithInteractive(true),
    pcg123.WithIdleTimeout(15*time.Minute),      // 图形生成可能需要更长时间
    pcg123.WithExecuteTimeout(60*time.Second),   // 复杂图表需要更长执行时间
)
defer cancel()

// 多步骤图形生成流程
// 第一步：数据准备
result1, err := executor.ExecuteCode(ctx, ...)

// 第二步：生成基础图表
result2, err := executor.ExecuteCode(ctx, ...)

// 第三步：生成高级分析图表
result3, err := executor.ExecuteCode(ctx, ...)
```

## 运行方式

```bash
# 设置环境变量
export PCG123_SECRET_ID="your-secret-id"

# 运行示例
cd 04_interactive_plot

```

## 交互式绘图优势

### 性能优化
```go
// 一次数据加载，多次使用
步骤1: 数据加载和预处理
步骤2: 基础图表 (使用步骤1的数据)
步骤3: 技术分析 (使用步骤1的数据 + 步骤2的计算)
步骤4: 综合仪表板 (使用所有前面的结果)
```

### 内存效率
- **变量复用**: 避免重复加载大型数据集
- **增量计算**: 基于已有结果进行进一步分析
- **状态保持**: matplotlib配置在整个会话中保持

### 开发效率
- **迭代优化**: 可以逐步完善图表样式
- **快速调试**: 中间结果立即可见
- **模块化**: 每个步骤相对独立但相互关联
