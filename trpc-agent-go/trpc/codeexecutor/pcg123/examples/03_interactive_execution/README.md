# 💬 交互式代码执行示例

**演示** 如何使用PCG123代码执行器的交互式模式，在同一个会话中执行多轮代码。

## 展示内容

- 🔄 **状态保持**: 变量和导入在多次执行间保持
- 📈 **渐进开发**: 逐步构建复杂的数据分析流程
- 💾 **会话管理**: 合理的超时和资源管理
- 🧪 **迭代调试**: 在同一环境中测试和修改代码

## 核心代码

```go
// 创建交互式执行器
executor, cancel, err := pcg123.NewCodeExecutor(conf,
    pcg123.WithInteractive(true),                // 启用交互式模式
    pcg123.WithIdleTimeout(10*time.Minute),      // 设置空闲超时
    pcg123.WithExecuteTimeout(30*time.Second),   // 设置执行超时
)
defer cancel()

// 第一步：导入库和初始化数据
result1, err := executor.ExecuteCode(ctx, ...)

// 第二步：使用第一步的变量
result2, err := executor.ExecuteCode(ctx, ...)

// 第三步：进一步处理数据
result3, err := executor.ExecuteCode(ctx, ...)
```

## 运行方式

```bash
# 设置环境变量
export PCG123_SECRET_ID="your-secret-id"
export PCG123_SECRET_KEY="your-secret-key"

# 运行示例
cd 03_interactive_execution
go run main.go
```

## 交互式模式特点

### 状态保持
- **变量持久**: 所有变量在会话期间保持
- **导入保持**: 库导入只需执行一次
- **环境一致**: 整个会话使用同一个Python环境

### 性能优化
- **减少重复**: 避免重复导入和数据加载
- **增量处理**: 可以逐步构建复杂分析
- **调试友好**: 可以随时检查中间状态

### 超时管理
```go
// 执行超时: 单次代码执行的最大时间
WithExecuteTimeout(30*time.Second)

// 空闲超时: 会话无活动时的保持时间
WithIdleTimeout(10*time.Minute)
```
